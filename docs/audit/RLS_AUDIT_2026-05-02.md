## Categorized

### Real exposures: 0
None of the SR=anon results expose private user data. The three "everyone can read" cases are all PUBLIC-BY-DESIGN catalogs:
- `agent_trust_scores` — public reputation score is the entire point of the trust-score subsystem
- `capability_registry` — public capability catalog (the "Yelp for AI services" data layer)
- `federation_origins` — federation peer list; intentional to allow clients to discover federated Settle instances

(If business decides any of these should be private, add an explicit RLS policy that enforces `authority_pubkey = auth.jwt() -> 'wallet_pubkey'`. Today, all three are intentional.)

### RLS verified working: 6
For these tables anon sees fewer rows than service_role — proof RLS is filtering:
- `auto_refill_queue`, `auto_refill_rules`, `gift_sends`, `kernel_receipt_attestations`, `phase5_executions`, `round_up_queue`

### Bug found and fixed (in-tree): 4
**group_account_members + group_accounts + group_spend_approvals + group_spend_requests**.

`infra/supabase/migrations/0046_rls_unprotected_tables.sql` defined policies that subquery the same table they protect:

```sql
create policy group_members_co_read on public.group_account_members
    for select using (
        group_id in (
            select group_id from public.group_account_members  -- ← recursion
            where member_pubkey = coalesce(auth.jwt()->>'wallet_pubkey', '')
        )
    );
```

When Postgres evaluates the policy, it queries the table, which triggers policy evaluation again. Postgres detects the loop and returns `500 infinite recursion detected in policy for relation "group_account_members"`. Effect: the four group tables are entirely **unreadable** from the application via the public REST API — the Groups feature can only run via service-role bypass, which is exactly what `apps/web/app/api/group-accounts/*` does.

**Fix:** `infra/supabase/migrations/0049_fix_group_rls_recursion.sql` extracts the membership lookup into a `SECURITY DEFINER` helper `public.user_group_ids()` that bypasses RLS for the inner query, then rewrites the four policies to call the helper. Standard Supabase pattern for "policy needs to read its own table."
